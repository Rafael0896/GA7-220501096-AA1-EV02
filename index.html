Estándares de Codificación

Lenguajes utilizados en el proyecto

  • HTML (HyperText Markup Language): Define la estructura y contenido de la página web utilizando HTML5 con etiquetas semánticas.
  • CSS (Cascading Style Sheets): Define la presentación visual incluyendo colores, tipografía, espaciado, y diseño responsive.
  • JavaScript: Añade interactividad a la página, maneja eventos y mejora la experiencia de usuario.

Estándares HTML

  • Declaración DOCTYPE: Siempre incluir para asegurar la interpretación correcta del HTML5.
  • Etiqueta lang: Especificar el idioma del documento para accesibilidad y SEO.
  • Meta tags: Incluir meta etiquetas esenciales para codificación y visualización.
  • Estructura semántica: Usar etiquetas semánticas en lugar de divs genéricos.
  • Indentación: 4 espacios para cada nivel de anidamiento.
<!DOCTYPE html>
<html lang="es">
    <head>
        <meta charset="UTF-8">
        <meta name="viewport" content="width=device-width, initial-scale=1.0">
    </head>

Estándares CSS

  • Variables CSS: Utilizar para valores reusables como colores y tamaños.
  • Organización: Agrupar estilos relacionados y comentar secciones principales.
  • Media Queries: Implementar para asegurar diseño responsive.
  • Selectores: Mantener selectores simples y evitar excesiva especificidad.
  • Reset/Normalize: Incluir reset CSS para consistencia entre navegadores.
:root {
    --color-primary: #3498db;
    --color-secondary: #2ecc71;
}

/* Estilos para navegación */
.nav-item {
    margin-left: 30px;
}

Estándares JavaScript

  • "use strict": Usar modo estricto para evitar errores comunes.
  • Programación orientada a objetos: Utilizar clases para organizar el código.
  • Event listeners: Usar addEventListener en lugar de atributos HTML.
  • Contexto (this): Mantener el contexto correcto con .bind() o arrow functions.
  • DOMContentLoaded: Inicializar scripts cuando el DOM esté cargado.
"use strict";

class NavManager {
    constructor() {
        this.init();
    }

    init() {
        const navLinks = document.querySelectorAll('.nav-link');
        navLinks.forEach(link => {
            link.addEventListener('click', this.handleNavClick.bind(this));
        });
    }
}

Nomenclatura para variables y constantes

  • HTML - IDs: Usar camelCase o kebab-case consistentemente: mainContainer o main-container
  • HTML - Clases: Usar kebab-case: feature-card, nav-item
  • CSS - Variables: Usar kebab-case con prefijo --: --color-primary, --font-size-heading
  • JavaScript - Variables: Usar camelCase: let userName = 'Juan'
  • JavaScript - Constantes: Usar camelCase para instancias: const navManager = new NavManager()
  • JavaScript - Constantes globales: Usar UPPER_SNAKE_CASE: const API_KEY = 'abc123'

Nomenclatura para clases, métodos y comentarios

  • Clases: PascalCase (primera letra mayúscula): class NavManager {}
  • Métodos: camelCase, verbos o frases verbales: handleClick() {}
  • Comentarios HTML: <!-- Encabezado principal del sitio -->
  • Comentarios CSS: /* Estilos para la navegación principal */
  • Comentarios JavaScript:
/**
* Maneja el evento de clic en los enlaces de navegación
* @param {Event} event - El evento del navegador
* @return {void}
*/
handleNavClick(event) {
    // Implementación...
}

Buenas prácticas implementadas

  • Separación de responsabilidades: HTML para estructura, CSS para estilos, JavaScript para comportamiento.
  • Código limpio: Siguiendo principios de legibilidad y mantenibilidad.
  • Responsive Design: Media queries para asegurar buena experiencia en todos los dispositivos.
  • Accesibilidad: Etiquetas semánticas y contraste de color adecuado.
  • Rendimiento: Optimización de recursos y estructura CSS eficiente.
  • Control de versiones: Organizar commits con mensajes descriptivos.
